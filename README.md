# Киноаук Максима и Оли

Домашнее локальное веб-приложение для выбора фильма на вечер. Проект выглядит как VHS-аукцион: колесо выбирает один фильм, маскот комментирует результат, победитель появляется на кассете, а импорт подтягивает списки "Буду смотреть" с Кинопоиска.

## Что умеет

- Импортирует публичные списки Кинопоиска:
  - `https://www.kinopoisk.ru/user/14758743/movies/planned-to-watch/`
  - `https://www.kinopoisk.ru/user/33826991/movies/planned-to-watch/`
- Собирает название, ссылку, постер, год и жанр.
- Дедуплицирует фильмы из двух списков.
- Крутит одно колесо и выбирает победителя случайно.
- Показывает победителя в VHS-кассете.
- Показывает победителя в fullscreen-модалке поверх колеса.
- Включает жанровые реакции маскота.
- Поддерживает «Жанровый аукцион»: по умолчанию колесо содержит весь каталог, а при выборе одного или нескольких жанров временно собирается только из подходящих фильмов. Общий список справа при этом не скрывается.
- Позволяет Максиму и Оле поставить по одной ставке в 10% на следующий аукцион; текущая вероятность каждого фильма видна прямо в списке.
- Для жанров включает разные премьеры:
  - ужасы: "КАССЕТА УЖАСА", растровый кровавый VHS-оверлей, затемнение, крик;
  - драма: дождь и драматическая мелодия;
  - боевик: два попадания с осыпанием стекла поверх экрана и реальные выстрелы;
  - комедия: смешные всплывающие элементы и boing/pop звук;
  - default: VHS static и штамп "СЕГОДНЯ В ПРОКАТЕ".
- Работает локально через Node.js без внешних npm-зависимостей.

## Быстрый запуск

Через ярлык или вручную:

```powershell
start-kinoauk.cmd
```

Или напрямую:

```powershell
node server.js
```

Открыть:

```text
http://127.0.0.1:5173
```

Остановить сервер:

```powershell
stop-kinoauk.cmd
```

Также можно нажать кнопку питания в правом верхнем углу сайта.

## Команды разработки

```powershell
npm test
npm run check
npm start
```

`npm run check` проверяет синтаксис `app.js`, `core.js`, `server.js` и запускает тесты.

## Правила работы над проектом

- Перед каждой задачей читать `docs/AI_CONTEXT.md` и учитывать уже принятые продуктовые решения.
- Стиль приложения является частью функциональности. На визуале, атмосфере, звуке, маскоте и VHS-вайбе не экономить.
- Если есть выбор между "быстро поставить заглушку" и сделать аккуратный стильный ассет/эффект, выбирать аккуратный вариант или честно зафиксировать временность решения.
- Любое изменение интерфейса проверять не только по тестам, но и глазами: текст не должен наезжать, элементы не должны выглядеть случайными, эффект должен попадать в домашний VHS-стиль.

## Как работает импорт

Кнопка "Обновить с Кинопоиска" вызывает `GET /api/refresh`.

Сервер:

1. Загружает публичные страницы списков.
2. Обходит страницы `page=1..10`.
3. Парсит ссылки `/film/<id>/` и `/series/<id>/`.
4. Достаёт:
   - `title`
   - `url`
   - `poster`
   - `year`
   - `genre`
5. Возвращает JSON в браузер.

Жанр и год берутся прямо из HTML списка, например:

```html
alt="Пила. 2004, ужасы"
<span>2004, ужасы</span>
```

Поэтому импорт почти не замедляется: не нужно заходить на страницу каждого фильма.

## Как работает выбор победителя

В `spin()` победитель выбирается заранее из тех же весов, которые нарисованы на колесе:

```js
const odds = calculateMovieOdds(auctionMovies(), state.stakes);
const winner = pickMovieByOdds(odds, Math.random);
```

После этого колесо анимируется так, чтобы остановиться на выбранном фильме. Без ставок все фильмы в текущем аукционе равновероятны; со ставками размер каждого сектора соответствует его реальному шансу.

## Жанровый аукцион

Жанры формируются из уже импортированных данных Кинопоиска. Режим начинается с «Весь каталог»; выбор нескольких жанров работает как «или», поэтому фильм участвует, если у него совпал хотя бы один жанр. Картины без жанра остаются в общем каталоге, но не попадают в конкретные жанровые выборки.

Изменение фильтра не запускает новый импорт. В списке участники текущего аукциона поднимаются наверх, а остальные фильмы остаются доступными ниже разделителя «Вне жанрового аукциона». «Жанры» находятся отдельной вкладкой рядом со «Списком» и «Импортом», поэтому настройки не меняют геометрию колеса. При нажатии жанра проигрывается короткая тематическая мини-сцена и маскот комментирует выбор. Для ужасов тень проходит по нижней части экрана как потерянный VHS-кадр, для остальных жанров есть соответствующий световой или плёночный жест. При `Применить` колесо делает короткий перемонтаж: прежние сектора уходят в центр, а новый набор разворачивается обратно. После запуска колеса выбор жанров блокируется вместе с остальным управлением.

## Ставки На Кассету

У Максима и Оли есть по одной ставке на следующий запуск: кнопки `М` и `О` рядом с фильмом добавляют ему по 10 процентных пунктов к обычному шансу. После прибавки оставшаяся вероятность поровну делится только между фильмами без ставок. При совпавшей ставке один фильм получает базовый шанс плюс 20 пунктов. Колесо рисует эти реальные доли и мягко перемонтируется при каждой изменённой ставке.

Процент каждого фильма показан в списке. Ставка действует только на следующий запуск, не работает для фильма вне текущего жанрового аукциона и автоматически очищается после результата.

## Звуки

Все базовые звуки генерируются через WebAudio: VHS-щелчки, hiss, rewind, tracking, дождь и комедийные эффекты.

Для horror-победы приложение дополнительно пытается проиграть реальный файл:

```text
assets/sounds/female-scream-pixabay-41894.mp3
```

Источник выбранного эффекта: [Loud Female Scream на Pixabay](https://pixabay.com/sound-effects/people-loud-female-scream-41894/), автор `Yin_Yang_Jake007 (Freesound)`, лицензия [Pixabay Content License](https://pixabay.com/service/license-summary/). Если файла нет или браузер не смог его проиграть, включается синтетический fallback.

Для боевика приложение проигрывает два реальных выстрела из `assets/sounds/action-gunshots.mp3`. Без неё остаётся только тихая механика VHS, без имитации выстрелов. Первый оверлей осыпающегося стекла появляется сразу, второй через 280 мс; оба берутся из `assets/action-glass-fall.png`, полностью исчезают за 1.45 секунды и не блокируют кнопки.
