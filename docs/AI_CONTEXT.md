# AI Context: Киноаук Максима и Оли

Этот файл нужен, чтобы будущая AI-сессия быстро поняла проект без перечитывания всей истории.

## Обязательное правило для AI

Перед началом любой задачи сначала прочитать этот файл и держать его решения в голове. Не начинать правки вслепую.

## Протокол работы с бэклогом

Это правило обязательно для любой AI-сессии, которая меняет проект.

1. Сначала прочитать этот файл, `docs/ARCHITECTURE.md` и `docs/ROADMAP.md`.
2. В `docs/ROADMAP.md` взять **только первую** задачу со статусом `READY` и с выполненными зависимостями. Не выбирать более интересную задачу ниже по списку и не объединять несколько задач в одну правку.
3. Если следующая задача имеет статус `WAITING_FOR_APPROVAL`, не писать код: кратко показать пользователю указанное в задаче решение, макет или технический выбор и дождаться явного согласования.
4. Перед началом работы перевести выбранную задачу в `IN_PROGRESS`, записать дату и короткое допущение, если оно было сделано.
5. После работы обновить в задаче: статус, фактическое решение, затронутые файлы, проверки и всё, что должен знать следующий агент. Если выбор остаётся за пользователем, вернуть задачу в `WAITING_FOR_APPROVAL`.
6. Не менять утверждённые продуктовые решения и не «улучшать по пути» соседние задачи без отдельного согласования. Новая идея сначала попадает в бэклог, а не в код.

Этот файл — краткая память проекта. `docs/ROADMAP.md` — порядок работы и журнал решений. После заметного изменения обе записи должны оставаться правдивыми, чтобы новая сессия не начинала проект заново в голове.

Стиль приложения очень важен: это не вторичная оболочка, а часть продукта. На визуале, звуке, маскоте, VHS-атмосфере и деталях победных эффектов не экономить. Быстрые заглушки допустимы только как явно временное решение; по умолчанию нужно делать аккуратно, цельно и с проверкой глазами.

## Цель продукта

Локальное приложение для Максима и Оли, которое выбирает фильм на вечер из двух публичных списков Кинопоиска "Буду смотреть". Атмосфера: домашний VHS, диктатура выбора, маскот-комментатор, кассета победителя.

Это личный локальный проект, не коммерческий сервис.

## Текущие продуктовые решения

- Один общий список фильмов и одно колесо.
- Будущая функция TMDb — не расширение текущего жанрового аукциона. В приложении будут два самостоятельных режима: **«Домашняя полка»** (нынешнее колесо и личный список) и **«Прокат по запросу»** (поиск по каталогу TMDb, полный пул и адаптивный способ выбора). Переключение между режимами не должно сбрасывать личный список, историю или ставку; активный режим всегда должен быть понятен визуально.
- Первый «Прокат по запросу» ищет **только в TMDb**. Он не должен пытаться искать по текущей домашней полке или будущей видеотеке.
- «Наша видеотека» — отдельная будущая функция: два списка Кинопоиска «Просмотрено», отдельно для Максима и Оли. Её нужно проектировать отдельной задачей, не смешивая с первым TMDb-прокатом.
- В «Прокате» по умолчанию ищем только фильмы. Сериалы добавляются явным чекбоксом «Учитывать сериалы» и смешиваются с фильмами в одном пуле, но каждый элемент обязан хранить `media_type`.
- Фильтр по актёру означает только актёрское участие (`cast`). Поиск по режиссёру — отдельный фильтр, добавлять постепенно. Человек для фильтра должен быть выбран явно через подсказки TMDb; не собирать полный пул по свободному тексту вроде «Питт».
- В данных для «Проката» сразу закладываем год, рейтинг, страну, длительность и историю сеансов, но UI включаем постепенно. История запросов полезна, но не первый приоритет.
- В режиме «Прокат» форма «Собрать вечер» подключена к серверу: жанры загружаются из TMDb, актёр/актриса ищется через подсказки TMDb, чекбокс «Учитывать сериалы» участвует в запросе, кнопки «Собрать кассеты» / «Очистить» работают, правая панель показывает реальный сохранённый пул. В UI не должно быть захардкоженных имён актёров или фильмов-заглушек.
- TMDb credential — только серверная конфигурация (`TMDB_API_TOKEN` / `TMDB_API_KEY` в окружении хостинга или локальном `.env`). Пользовательский UI не должен просить, показывать или сохранять ключи. Если сервер не настроен, «Прокат» показывает недоступное состояние и ждёт настройки backend.
- У Максима и Оли есть отдельные ставки на следующий запуск: жетоны `М` и `О` добавляют выбранному фильму по 10 процентных пунктов к его обычному базовому шансу. После этого остаток честно и поровну перераспределяется между фильмами без ставок; совпавшие ставки складываются в +20 пунктов. Шанс показывается у каждого фильма, а размер сектора колеса совпадает с этим шансом. При каждой ставке колесо делает тот же спокойный перемонтаж, что и при применении жанрового фильтра. Ставки действуют только в текущем жанровом пуле и очищаются после результата.
- «Жанровый аукцион» является дополнительным режимом: по умолчанию в колесе весь каталог; после выбора и применения жанров колесо временно использует подходящие фильмы. В полном списке справа участники активного аукциона показываются первыми, а остальные уезжают вниз за разделитель «Вне жанрового аукциона». Несколько жанров работают по «или». «Жанры» живут отдельной вкладкой рядом со «Списком» и «Импортом», поэтому не меняют геометрию колеса. У каждого жанра есть короткая тематическая сценка и реплика маскота, а применение запускает спокойный canvas-перемонтаж без тряски. Для horror используется краткий слой `assets/genre-horror-shadow.png`, который проходит по нижней части экрана; отдельная сценка триллера отключена. Облачко маскота комиксно всплывает на каждой новой реплике. Панель ограничивает высоту списка жанров, чтобы не ломать композицию на невысоких экранах.
- При применении жанра «Драма» на 2.3 секунды запускается сдержанный canvas-слой «дождь на стекле»: цельная фоторамка из `assets/effects/drama-rain-frame-v2.png` оставляет центр экрана почти чистым, а капли и тонкие потёки живут только по периметру. Слой не ловит клики, тихо смещается вниз, удаляется после анимации и не запускается при простом выборе чипа.
- Дневной и ночной фоны комнаты не должны получать CSS-фильтры, blur или scanline-оверлеи: `.poster-backdrop` и `body::after` оставляем чистыми, чтобы не появлялись рябь и смазывание поверх самих изображений.
- Актуальная пара фонов комнаты: `assets/backdrop/vhs-night-lounge.png` и `assets/backdrop/tv-day-lounge.png`. Это 2K-композиты с тёмной VHS-комнатой, большим окном, CRT-шумом, настоящим маскотом на тумбочке, небрежно наклеенными постерами «Криминального чтива» и «От заката до рассвета». Не заменять их на CSS-коллаж и не добавлять поверх них рябь/блюр. В дневной и ночной версии вид за окном должен быть 1 в 1 по композиции: тот же соседский дом, улица, деревья и ракурс; меняется только день/ночь, свет и атмосфера.
- Разделение "фильмы Максима / фильмы Оли" пока не делаем.
- Результат выбирается случайно, после старта переголосовки нет.
- Салют/фейерверки удалены, вместо них VHS static burst и штамп.
- Звук должен быть не казино, а механика VHS: тихий мотор ленты, мягкий hiss, tracking и rewind. Прокрутка колеса не использует частые высокие щелчки: Оле от них болит голова. Во время вращения звучит спокойный мотор, а в конце допускаются только редкие низкие ролики.
- После выбора всегда звучит общий очень тихий тёплый VHS-аккорд, независимо от жанра; резкие glitch, click и hiss-всплески для этой общей победной реплики запрещены. Жанровые звуки накладываются поверх него отдельно.
- Любая обычная кнопка интерфейса должна давать один мягкий низкий отклик. Это делегированный обработчик на документе, поэтому он покрывает и динамические кнопки карточек. Кнопки со своим звуковым сценарием (`Крутить`, звук, жанровые чипы, ставки и применение фильтра) исключены, чтобы не создавать двойной звук.
- Звук сужения/перемонтажа колеса должен быть спокойным: короткое приглушённое втягивание ленты вместо резкого rewind. Он одинаково используется при жанровом фильтре и при ставке.
- Маскот должен быть странным, смешным и домашним, но не перекрывать текст.
- Horror-жанр должен давать отдельную fullscreen-премьеру: затемнение, растровый blood overlay `assets/blood-overlay.png`, "КАССЕТА УЖАСА", женский scream sound при наличии файла.
- Боевик даёт fullscreen-премьеру только с двумя последовательными попаданиями и осыпанием стекла `assets/action-glass-fall.png`: второй оверлей приходит через 280 мс после первого, эффект лежит поверх модалки, не ловит клики и исчезает за 1.45 секунды. Старые трассеры, вспышки, статичная трещина, VHS-штамп и synthetic gunshots для боевика удалены. Два реальных выстрела подключаются файлом `assets/sounds/action-gunshots.mp3`; без файла остаётся только тихая механика VHS.
- Победитель теперь показывается в fullscreen-модалке поверх колеса. Нижняя кассета остаётся вторичным резюме.

## Основные файлы

- `server.js`: локальный HTTP-сервер, `/api/refresh`, `/api/shutdown`, парсер Кинопоиска.
- `rental-db.js`: локальная SQLite-схема будущего «Проката» через встроенный `node:sqlite`; создаёт `data/kinoauk.sqlite`, накатывает миграции v1, но пока не подключён к UI и TMDb-сети.
- `tmdb-client.js`: серверный TMDb-клиент для «Проката»; читает `TMDB_API_TOKEN`, `TMDB_READ_ACCESS_TOKEN`, `TMDB_BEARER_TOKEN`, `TMDB_TOKEN` или `TMDB_API_KEY` из окружения/локального `.env`, нормализует поиск людей и сохраняет кандидатов в `people`.
- `server.js` уже умеет `POST /api/rental/sessions` и `GET /api/rental/sessions/:id`: сервер собирает полный пул TMDb, сохраняет `rental_sessions` / `rental_session_items` и читает их обратно.
- `server.js` уже умеет `POST /api/rental/sessions/:id/select`: выбор идёт равновероятно из сохранённого полного пула, сохраняет выбранную кассету и помечает режим `wheel` или `vhs_machine`.
- `core.js`: чистая логика для тестов и браузера.
- `app.js`: DOM, canvas wheel, localStorage, звук, маскот, VHS-эффекты.
- `styles.css`: вся визуальная система.
- `assets/sounds/README.md`: источник и лицензия реального horror scream.
- `test/`: тесты на core, серверный парсер и базовую структуру проекта.

## Данные фильма

Фильм в приложении:

```js
{
  title: "Пила",
  url: "https://www.kinopoisk.ru/film/64187/",
  poster: "https://...",
  year: "2004",
  genre: "ужасы"
}
```

Старые локальные фильмы могут не иметь `year/genre`; следующий импорт с Кинопоиска дозаполнит данные.

## Стиль будущих изменений

- Не превращать главный экран в landing page.
- Не добавлять внешние зависимости без явной нужды.
- Не ломать локальный запуск через `start-kinoauk.cmd`.
- Не делать сетевой импорт медленнее без причины.
- Все новые чистые функции сначала рассмотреть для `core.js`.
- Новую функциональность по возможности покрывать тестами.
- Для «Проката» локальная БД уже подготовлена в `rental-db.js`. Не добавлять SQLite npm-зависимость без отдельной причины: текущий локальный Node 24 поддерживает `node:sqlite`. `.env` и `data/*.sqlite` должны оставаться вне git.
- `GET /api/rental/people?q=...` уже подключён в `server.js`. Он ищет только персон TMDb, сохраняет кандидатов в `people` и не собирает пул фильмов. Без `TMDB_API_TOKEN` возвращает понятную ошибку `TMDB_TOKEN_MISSING`.
- `GET /api/rental/genres?mediaType=movie|tv` уже подключён в `server.js`. Он кэширует жанры TMDb в `genres`, держит `movie` и `tv` раздельно даже при одинаковом `tmdb_id`, не создаёт `media` и без ключа возвращает `TMDB_TOKEN_MISSING`.
- `POST /api/rental/sessions` уже собирает полный пул по фильтрам и сохраняет его в SQLite. Если выбран актёр/режиссёр, сервер берёт полные person credits и фильтрует локально; если человека нет, идёт по discover-страницам до конца. `GET /api/rental/sessions/:id` возвращает сохранённый полный пул обратно.
- `POST /api/rental/sessions/:id/select` уже выбирает победителя из сохранённого полного пула. До 24 кассет `selectionMode` = `wheel`, больше 24 = `vhs_machine`. UI-анимация для этого выбора ещё не подключена.
- UI «Проката» использует живые элементы `rentalGenreSelect`, `rentalPersonInput`, `rentalIncludeTv`, `rentalBuildPoolButton`, `rentalClearButton`, `rentalTapeList`. При дальнейших правках не возвращать hardcoded-персон/фильмы и не смешивать его с текущей «Полкой».
- Не возвращать сильную тряску победителя: пользователь сказал, что от неё тошнит.
- Для реального женского крика использовать `assets/sounds/female-scream-pixabay-41894.mp3`; если файла нет, текущий WebAudio fallback допустим.
