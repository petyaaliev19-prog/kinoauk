# AI Context: Киноаук Максима и Оли

Этот файл нужен, чтобы будущая AI-сессия быстро поняла проект без перечитывания всей истории.

## Цель продукта

Локальное приложение для Максима и Оли, которое выбирает фильм на вечер из двух публичных списков Кинопоиска "Буду смотреть". Атмосфера: домашний VHS, диктатура выбора, маскот-комментатор, кассета победителя.

Это личный локальный проект, не коммерческий сервис.

## Текущие продуктовые решения

- Один общий список фильмов и одно колесо.
- Разделение "фильмы Максима / фильмы Оли" пока не делаем.
- Результат выбирается случайно, после старта переголосовки нет.
- Салют/фейерверки удалены, вместо них VHS static burst и штамп.
- Звук должен быть не казино, а механика VHS: щелчки, hiss, tracking, rewind.
- Маскот должен быть странным, смешным и домашним, но не перекрывать текст.
- Horror-жанр должен давать отдельный страшный reveal.

## Основные файлы

- `server.js`: локальный HTTP-сервер, `/api/refresh`, `/api/shutdown`, парсер Кинопоиска.
- `core.js`: чистая логика для тестов и браузера.
- `app.js`: DOM, canvas wheel, localStorage, звук, маскот, VHS-эффекты.
- `styles.css`: вся визуальная система.
- `test/`: тесты на core, серверный парсер и базовую структуру проекта.

## Данные фильма

Фильм в приложении:

```js
{
  title: "Пила",
  url: "https://www.kinopoisk.ru/film/64187/",
  poster: "https://...",
  year: "2004",
  genre: "ужасы"
}
```

Старые локальные фильмы могут не иметь `year/genre`; следующий импорт с Кинопоиска дозаполнит данные.

## Импорт Кинопоиска

Списки зашиты в `server.js`:

- `https://www.kinopoisk.ru/user/14758743/movies/planned-to-watch/`
- `https://www.kinopoisk.ru/user/33826991/movies/planned-to-watch/`

Парсер берёт жанр и год из HTML списка, не ходит на страницу каждого фильма. Это важно для скорости.

## Проверки

Перед финалом после изменений запускать:

```powershell
npm run check
```

Если менялся только серверный парсер, достаточно:

```powershell
node --check server.js
npm test
```
